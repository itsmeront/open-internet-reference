# Technical Experts

Engineers, cryptographers, and interdisciplinary scholars who frequently explain how the Internet and security systems actually work — useful for expert consultation during litigation, regulatory proceedings, or technical due diligence.

These are **not attorneys**. Pair them with counsel from the [Lawyers](lawyers.md) page and organizations from [Organizations](organizations.md).

## When to Use Technical Experts vs. Lawyers

| Need | Start with |
|------|------------|
| Explain how DNS, routing, or protocols work | Internet architecture experts (Cerf, Reed, Vixie, Weinstein) |
| Explain end-to-end design or ISP filtering tradeoffs | Reed, Weinstein, Cerf |
| Explain software architecture, OOP, or GUI prior art | Alan Kay |
| Explain encryption feasibility or key management | Cryptography experts (Diffie, Zimmermann, Schneier, Blaze) |
| Explain platform liability or regulatory tradeoffs | Law/policy technologists (Zittrain, Lessig) |
| File motions, represent you in court, privilege | [Lawyers](lawyers.md) |
